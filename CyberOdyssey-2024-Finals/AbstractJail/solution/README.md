In this challenge, we are given a Python script designed to audit the executed code. If it detects any banned AST nodes or critical functions and commands, it will terminate the execution.

To solve this challenge, we need to bypass two restrictions: the first is avoiding the use of banned AST nodes, and the second is circumventing the audit hook by not using critical functions.

To begin, we must find a way to bypass the audit hook. The code indicates that the audit is not performed for the first time (it uses `first_audit` as a flag). Therefore, we can set `first_audit` to `True` to bypass the audit hook in the next execution. While this may seem straightforward, we cannot directly set `first_audit` to `True` because [`ast.Assign`](https://docs.python.org/3/library/ast.html#ast.Assign) is banned.

Since `first_audit` is a global variable, we can use the [`globals()`](https://docs.python.org/3/library/functions.html#globals) function to access global variables and set `first_audit` to `True` using the dunder method `__setitem__`.

```py
getattr(globals(), '__setitem__')('first_audit', True)
```

The `True` value is an [`ast.Constant`](https://docs.python.org/3/library/ast.html#ast.Constant), so we cannot use it directly. Instead, we can use the [`callable()`](https://docs.python.org/3/library/functions.html#callable) function, which returns a Boolean value for a callable function, to obtain the `True` value.

```py

```
